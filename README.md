# Lyman-alpha Bubble Size Recovery

CNN pipeline to recover the ionized bubble size distribution during reionization
from Lyman-alpha spectra of high-redshift LAEs.

**Supervisor:** Anson D'Aloisio &nbsp;|&nbsp; **Author:** Aryana Haghjoo

---

## Project Goal

Train a 1-D CNN that maps an observed Lya spectrum → ionized bubble size,
then apply it to real JADES z~7–8 galaxies to recover the bubble size
distribution and compare to simulation.

## Data

### Simulation (from Anson)
RT snapshots in `data/for_aryana/late_end_early_start/`:
- `RTcubes/gas_z=*`: 200³ × 21-field binary float32 (1 header float + data)
- `Lya_transmission/Lya_transmission.z=*`: 10 000 × 2 000 float32 spectra
- `Lya_transmission/sl_params_z=*`: sightline params (start_inds, angles, halo_mass)
- `Lya_transmission/wav_out`: 2 000 wavelength bins in Å (descending)

Field index 15 (`fion`) is the ionised fraction used for bubble size labels.

### JADES DR4 (observational)
JWST/NIRSpec prism spectra — same instrument for templates and evaluation:
- **Templates** z=4–5.5 (IGM transparent → intrinsic line shape): 325 objects
- **Evaluation** z=7–8 (reionization era): 108 objects

## Quick Start

### 1. Prepare the simulation dataset
```bash
# Compute RT-cube multi-ray MFP labels for all available redshifts
python3 scripts/task1_transmission_to_bubble_size/prepare_dataset.py

# Generate mock Lya lines (JADES template × transmission)
python3 scripts/task3_mock_lya_lines/prepare_mock_dataset.py
```

### 2. Train models
```bash
# Task 1: raw transmission → bubble size
python3 scripts/task1_transmission_to_bubble_size/train_model.py

# Task 4: mock Lya line → bubble size
python3 scripts/task4_train_mock_lines/train_model.py
```

### 3. Evaluate on real JADES galaxies
```bash
python3 scripts/task6_evaluate_jades/evaluate.py
```

### 4. View results in notebooks
Open any notebook in `notebooks/` — they load pre-computed results and
do not require re-running the training.

## Multi-Redshift Ready

All scripts automatically detect and process every redshift snapshot
present in the data directories:
- `prepare_dataset.py` → one `dataset_z*.npz` per snapshot
- `prepare_mock_dataset.py` → one `mock_z*.npz` per snapshot
- `train_model.py` → loads all `dataset_z*.npz` / `mock_z*.npz` at once

**Adding a new redshift**: place the RT cube and transmission files in the
data directories, then re-run the preparation scripts. Training picks up
the new data automatically.
